## Pozorování

- **Nejsilnější moment sezení:** run objektivně vygeneroval učebnicovou smrtelnou
  spirálu (vynucená hlučná karta Zbrklostí → Žár skok → zákaz Lesti → kolaps →
  léčka → TPK) — ale hráč ji **neviděl**, viz níže.
- **Nejslabší / mrtvý moment:** hráč nerozuměl resolučnímu systému („soubojový
  systém je buď primitivní, nebo netuším") — přitom systém funguje přesně dle
  pravidel. **Hra svá pravidla nevysvětluje**: UI čísla ukazuje, ale neučí
  souvislosti (co dělá prokletá karta právě teď, proč rostl Žár dvojnásobně,
  že rider je volba, co znamená postih za zranění dlouhodobě).
- **Kde se hráč ztratil:** kauzalita řetězců (vynucené zahrání kartou Zbrklost,
  zákaz tagu Roztřesenýma rukama, +1 z hlasu z auta) — události proběhly správně,
  ale bez vysvětlující stopy v UI.

## Ladění

- **Resoluční čísla:** run trval 5 uzlů a skončil TPK přes 5–6 pásmo (zranění
  každý uzel) + Brody × hlučné karty; ekonomika seděla (1 bedna přes rider).
  Solo/2p data — bez závěru k číslům.
- **Tempo: celý run ~5 minut** při svižném klikání (solo, bez čtení nahlas).
  Cíl 30 min předpokládá 4 hráče + stůl + protokoly nahlas; solo tempo je ale
  reálně 5–6× rychlejší, než metriky implicitně čekají.
- **Protokoly:** bez připomínek od hráče (fallback šablony).
- **Karty/uzly:** bez konkrétního nálezu.

## Závěr

Jedna věta: **před dalším sezením přidat do UI vysvětlující vrstvu pravidel**
(nápověda / průvodce prvním uzlem / „proč se to stalo" anotace u událostí typu
vynucená karta, zákaz tagu, rider, práh Žáru) — systém funguje, ale je pro
hráče neviditelný.
